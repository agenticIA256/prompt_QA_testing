# 🧭 Workflow / Steps
1. Resolve inputs & paths relative to working_directory.
2. Load drupal11_audit_report.md (error if empty).
3. Convert Markdown → Confluence Storage Format (XHTML)
  * Preserve headings, tables, lists, code blocks.
  * Mermaid: keep as fenced code block (```mermaid) or tag for later rendering (no native Confluence render).
4. Process attachments referenced in Markdown (path):
  * Resolve local paths under working_directory
  * Upload as Confluence attachments
  * Rewrite references in the body to Confluence attachment URLs.
5. Determine page title:
   * Use page_title if provided, otherwise: Drupal 11 Audit — <repo_name> — <commit_sha> — <timestamp> (fallback).
6. Create  the Confluence page in confluence_space_key (and parent_page_id if provided).
7. Write artifacts to working_directory/publisher/<timestamp>/:
   * confluence_publish_report.json (page id, URL, version, attachments, stats)
   * execution_log.json (append fields; do not erase upstream content)
8. Append Task Execution Report: API calls, files processed, timing, errors.
